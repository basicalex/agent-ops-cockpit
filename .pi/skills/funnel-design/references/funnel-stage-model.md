# Funnel Stage Model

Use this when a request needs a complete journey rather than a single page patch.

| Stage | User state | Information job | Typical surface | Primary CTA | Proof need |
|---|---|---|---|---|---|
| Signal Capture | Arrives with source intent | Match the promise that brought them here | landing route, campaign page, SEO page | Continue / primary conversion | Message-source match |
| Orientation | Asking “is this for me?” | Name audience, category, outcome | hero, intro, metadata | Book/demo/start/join | specific outcome |
| Problem/Need Fit | Recognizes pain/opportunity | Show you understand the job-to-be-done | problem section, use case | See how it works | examples |
| Product Promise | Evaluating value | State transformation and payoff | value prop, feature proof | Start path | measurable claim |
| Mechanism | Wants plausibility | Explain how it works simply | workflow, product visual | Try/demo | product evidence |
| Proof | Needs confidence | Prove claims near the claim | testimonials, logos, screenshots, case studies | Choose offer | concrete proof |
| Offer | Compares cost/risk | Package price, plan, demo, waitlist, or trial | pricing, plan cards | Pick plan/book | terms/fit clarity |
| Action | Ready but wary | Make the next action obvious and low-friction | CTA, form, checkout, auth | Submit/start/pay | trust/privacy cues |
| Capture/Qualification | Sharing info | Ask only what is needed | lead form, signup, quiz | Submit | why fields matter |
| Post-Conversion Momentum | Wants reassurance | Confirm next step and timeline | thank-you, calendar, email | Schedule/setup | confirmation |
| Activation | New user wants value | Guide to first value | onboarding, empty state, checklist | Complete first action | progress |
| Retention/Expansion | Has value or churn risk | Encourage repeat use and sharing | lifecycle, dashboard, referral | Return/upgrade/refer | saved progress/result |
