---
name: prd-dev
description: Draft and refine a Taskmaster PRD in .taskmaster/docs/prd.txt.
---

## Output
Create or update `.taskmaster/docs/prd.txt` as the canonical spec.

## Process
1. Ask clarifying questions to resolve gaps.
2. Draft sections: problem, goals, non-goals, user stories, requirements.
3. Include acceptance criteria and dependencies.
4. Cover risks, performance, security, UX, and maintainability.
5. Define a test strategy and success metrics.

## Format
Plain text with clear headings and bullet lists.
